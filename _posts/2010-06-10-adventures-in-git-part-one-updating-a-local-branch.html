I'm starting to use git after 2 years of SVN usage.

As I understand a common workflow is, as derived from the git tutorial: 

git clone 
git checkout -b myworkingbranch # create a new branch to add a feature or fix bug
# do some work add / edit files
git add files
git commit -m 'a commit message'
git checkout master
git merge myworkingbranch
... but what if you've already updated your master branch from the remote origin? or the remote origin has been updated? The answer is to update your branch by merging with the master or remote. e.g. 
git checkout master
git pull # git updates local files
git checkout myworkingbranch # branch is out of date
git merge master # git updates your branch
